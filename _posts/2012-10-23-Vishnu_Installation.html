Installation and launching

Requirements

VISHNU can be installed on Linux systems only (preferably Debian distributions), and requires the following software:

  • CMake (2.8.5 or greater)
  • Boost C++ libraries (1.46.1 or greater)
  • EMF4CPP (0.0.2 patched - see debian packages below - or greater)
  • SSH
  • PostGreSQL (8.0 or greater) or MySQL (5.0 or greater)
  • On all hosts providing task mgmt services (TMS module), one of the following libraries:
    • Torque API v3.2.6
    • IBM LoadLeveler 3.5.1.14
    • SLURM v2.2.1, 2.2.7, 2.3 or 2.4
    • LSF 7.0.6.134609
    • Grid Engine 2011.11
    • No batch scheduler

Launching the VISHNU platform

VISHNU is a distributed system and can therefore be installed on several systems as well as on a single system. When using several systems, the choice of which services to deploy on each system is dependent on the characteristics of the systems and of the network.

Here is the list of processes that must be started to deploy all VISHNU services (see also the "VISHNU Administrator's guide" in the Documentation section):

  • Database instance: either a MySQL or PostgreSQL database
  • Optionally the dispatcher process (proxy to VISHNU SeDs)
  • All VISHNU SeD processes (umssed, tmssed, imssed, fmssed)
