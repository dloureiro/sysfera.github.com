Here is the corresponding changelog:

  • New features:
    • Added beta support for submitting jobs to a Cloud, using DeltaCloud (tested with OpenStack only). Currently, only the Vishnu user is used, there is no user switch during submission. The credentials need to be provided to VISHNU though an configuration file, options, or environment variables.
    • Automtests have been updated: they do not launch the Vishnu platform, they use an already deployed one.
  • Bug fixes and improvements:
    • The compilation process has been updated: you can now compile the client CLI and the servers separately
    • Files necessary to build vishnu java library have been included in this repository. Currently there is no doc for this, see scripts/ALL_prepareScript.sh if you want the lib.
    • fixed a bug where signal-interrupted zmq calls (EINTR) were not handled properly
    • fixed a bug on the "POSIX" backend where the wallclock time was not correctly handled which caused submission scripts to sometimes be deleted before being executed.
  • Warnings:
    • The database needs to be updated using scripts in core/database/database_update_addvminfo_*.sql

You can download the tarball containing the sources on the dedicated Vishnu download Page or directly from the following link: Vishnu 3.0.0 beta1
